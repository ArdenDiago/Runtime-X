Extend internal/process/runner.go with SIGINT/SIGTERM interception, forwarding to the child process, and POSIX-correct exit code emulation for signal-killed children (128+N).

Purpose: Phase 1 isolated the child in its own process group (Setpgid: true) so the kernel does NOT auto-deliver Ctrl+C to the child. Explicit forwarding is therefore mandatory — without it, the child runs forever when the user presses Ctrl+C. This plan implements the forwarding and the correct POSIX exit codes.

  <name>Task 1: Add signal interception and forwarding to Run()</name>
  <files>internal/process/runner.go</files>
  <action>
Modify Run() in internal/process/runner.go to intercept SIGINT and SIGTERM, log receipt, forward to child, and wait for child to exit before returning. The existing doneCh goroutine pattern and Setpgid:true are already in place — only a sigCh setup and a select replacement are needed.

Step-by-step changes to Run():

1. Add "os/signal" to the import block (already imports "syscall" and "os/exec" — add "os/signal" and "errors" if not already present; "errors" is already imported).

2. After the `fmt.Fprintf(os.Stderr, "[rtx] spawned PID %d\n", cmd.Process.Pid)` line, add:
   ```go
   // SIG-04: buffered capacity 1 — signal.Notify does a non-blocking send
   sigCh := make(chan os.Signal, 1)
   signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM) // SIG-01, SIG-02
   defer signal.Stop(sigCh)
   ```
   Note: signal.Notify MUST be called AFTER cmd.Start() succeeds — before that, there is no child to forward to.

3. Keep the existing doneCh goroutine unchanged:
   ```go
   doneCh := make(chan error, 1)
   go func() { doneCh <- cmd.Wait() }()
   ```

4. Replace the bare `waitErr := <-doneCh` receive with a select:
   ```go
   var waitErr error
   select {
   case sig := <-sigCh:
       // LOG-02: log signal receipt before forwarding
       fmt.Fprintf(os.Stderr, "[rtx] received signal %s\n", sig)
       // SIG-03 + ERR-03: forward signal; swallow "already finished"
       if err := cmd.Process.Signal(sig); err != nil && !errors.Is(err, os.ErrProcessDone) {
           fmt.Fprintf(os.Stderr, "[rtx] signal forward failed: %v\n", err)
       }
       waitErr = <-doneCh // SIG-03: always wait — zombie prevention intact
   case waitErr = <-doneCh:
       // natural exit — no signal handling needed
   }
   ```

5. Update the resolveExitCode call to pass cmd.ProcessState:
   ```go
   code := resolveExitCode(waitErr, cmd.ProcessState)
   ```

IMPORTANT — Do NOT forward to the process group (do not use syscall.Kill(-pgid, sig)). cmd.Process.Signal(sig) targets the child PID, which is correct for a single-process runner. Setpgid:true means the child is isolated but cmd.Process.Signal() still reaches it by PID.

IMPORTANT — Do NOT call signal.Notify before cmd.Start() returns. Signals received before the child is running have no target.

<task type="auto">
  <name>Task 2: Extend resolveExitCode for 128+N POSIX exit codes</name>
  <files>internal/process/runner.go</files>
  <action>
Modify resolveExitCode() to accept a *os.ProcessState parameter and compute 128+N when the child was killed by a signal. Currently resolveExitCode(err error) returns -1 via ExitCode() for signal-killed children, which os.Exit(-1) converts to 255 on Linux — incorrect POSIX behavior.

Change the signature:
```go
func resolveExitCode(err error, state *os.ProcessState) int {
```

Inside the `if errors.As(err, &exitErr)` block, after `code := exitErr.ExitCode()`, add the 128+N check BEFORE the `return code` line:
```go
code := exitErr.ExitCode()
// EXIT-03: ExitCode() returns -1 for signal-killed processes.
// WaitStatus.Signaled() detects this and provides the actual signal number.
if code == -1 && state != nil {
    if ws, ok := state.Sys().(syscall.WaitStatus); ok {
        if ws.Signaled() {
            return 128 + int(ws.Signal())
        }
    }
}
return code
```

The state parameter comes from cmd.ProcessState, which is populated by cmd.Wait() before it sends to doneCh. Accessing cmd.ProcessState after `waitErr = <-doneCh` (in both select branches) is safe.

No new imports needed — syscall is already imported.

Signal-to-exit-code reference (for verification):
- SIGINT (2) -> 130
- SIGTERM (15) -> 143
- SIGKILL (9) -> 137 (rtx cannot forward SIGKILL, but child may receive from elsewhere)
