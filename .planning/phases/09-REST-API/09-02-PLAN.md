# Plan: Phase 09-02 - Lifecycle Handlers & CORS

## Goal
Complete all Phase 09 requirements (API-03 to API-09) by implementing process detail, update, delete, lifecycle (start/stop), logs, and CORS.

## Tasks

### 1. Implement Resource Handlers
- `GET /api/processes/{name}` (API-03): `GetProcess`.
- `PUT /api/processes/{name}` (API-04): `UpdateProcess`.
  - Must check that process is stopped before updating definition.
- `DELETE /api/processes/{name}` (API-05): `DeleteProcess`.

### 2. Implement Lifecycle & Log Handlers
- `POST /api/processes/{name}/start` (API-06): `StartProcess`.
- `POST /api/processes/{name}/stop` (API-07): `StopProcess`.
- `GET /api/processes/{name}/logs` (API-08): `GetLogs`.

### 3. Implement CORS Middleware (API-09)
- Simple CORS middleware allowing all origins, methods, and headers (v1.1 simplicity).

## Verification Strategy

### Automated Tests
- `internal/api/handlers_test.go`:
  - Test `GetProcess` (404 vs 200).
  - Test `StartProcess` and `StopProcess` (verify scheduler interactions).
  - Test `DeleteProcess` (verify error if running).
  - Test `GetLogs` returns expected envelope.
  - Test CORS headers presence in responses.

### Manual Verification
- `go test -v ./internal/api/...` must pass.
- `go build ./...` must exit 0.
