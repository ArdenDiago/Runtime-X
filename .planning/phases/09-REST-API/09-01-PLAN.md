# Plan: Phase 09-01 - API Foundation & Cleanup

## Goal
Restore clean build state by removing legacy `api-service/` and establishing the core REST API foundation (Server, Router, and Base Handlers) in `internal/api/`.

## Tasks

### 1. Cleanup Legacy Code
- `git rm -rf api-service/`
- Verify `go build ./...` succeeds after removal.

### 2. Implement API Server Core
- Create `internal/api/server.go`.
- Define `Server` struct with `Scheduler *scheduler.Scheduler`.
- Implement `NewServer(s *scheduler.Scheduler) *Server`.
- Implement `(s *Server) Routes() http.Handler` using `http.ServeMux` (Go 1.22+ patterns).
- Implement JSON envelope helper: `send(w http.ResponseWriter, status int, data any, err error)`.

### 3. Implement Base Handlers
- Implement `GET /api/processes` (API-01): `ListProcesses`.
- Implement `POST /api/processes` (API-02): `CreateProcess`.
- Use a `processJSON` struct to map between `scheduler.ProcessDef` and JSON (seconds for durations).

## Verification Strategy

### Automated Tests
- Create `internal/api/handlers_test.go`.
- Test `GET /api/processes` with empty and non-empty scheduler.
- Test `POST /api/processes` with valid and invalid payloads.
- Test `POST /api/processes` with invalid names (check slug regex integration).

### Manual Verification
- `go build ./...` must exit 0.
- `go test -v ./internal/api/...` must pass.
