# Plan: Phase 11-01 - React Scaffolding and Core Dashboard

## Goal
Scaffold the React application and build the core dashboard showing the process list with real-time (polling) status updates.

## Context
Phase 11 depends on Phase 10 (CLI Serve). The API must be running for the frontend to be developed and tested.

## Tasks

### 1. Scaffold React Application
- Run `npm create vite@latest web -- --template react-ts` from the project root.
- Install React 19 dependencies.
- Configure `vite.config.ts` with a proxy to `http://localhost:8080/api`.
- Clean up the default Vite template and set up global `index.css`.

### 2. Implement API Client and Types
- Create `web/src/api/types.ts` with interfaces matching Go structs.
- Create `web/src/api/client.ts` with a `fetch` wrapper for `GET /api/processes` and `POST /api/processes`.

### 3. Build Core Components
- Create `web/src/hooks/usePolling.ts` for consistent interval handling.
- Create `web/src/components/StatusBadge.tsx` for visual status indicators.
- Create `web/src/components/ProcessList.tsx` to display all registered processes.
- Integrate `usePolling` into `ProcessList` to refresh the list every 2 seconds.

### 4. Build Process Creation Form
- Create `web/src/components/ProcessForm.tsx`.
- Implement form handling for `name`, `command`, `args`, `restart policy`, and `dependencies`.
- Handle `POST /api/processes` on submission and refresh the list.

## Verification Strategy

### Automated Tests
- Since this is a UI prototype, focus on manual verification.
- Verify that `npm run build` succeeds and produces `web/dist`.

### Manual Verification
- Start `rtx serve`.
- Open the dashboard in the browser.
- Verify that processes registered via `curl` appear in the list.
- Create a new process via the form and verify it appears in the list.
- Verify that status updates (e.g., from `idle` to `running`) reflect in the UI automatically.
