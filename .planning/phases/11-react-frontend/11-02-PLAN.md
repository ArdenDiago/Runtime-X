# Plan: Phase 11-02 - Log Viewer and Process Controls

## Goal
Implement real-time log viewing and full control over process lifecycle (start, stop, delete) from the UI.

## Context
Phase 11 depends on Phase 10 (CLI Serve) and the core components from 11-01.

## Tasks

### 1. Implement Start/Stop Controls
- Add "Start" and "Stop" buttons to each item in `ProcessList`.
- Call `POST /api/processes/:id/start` and `POST /api/processes/:id/stop`.
- Update the process list after an action and let polling catch the state transition.

### 2. Implement Log Viewer
- Create `web/src/components/LogViewer.tsx`.
- Poll `GET /api/processes/:id/logs` every 2 seconds when a process is selected.
- Format log lines with timestamps (if provided) and stream tags (stdout/stderr).
- Ensure the viewer scrolls to the bottom on each poll update.

### 3. Implement Delete and Edit
- Add "Delete" button with a confirmation dialog.
- Call `DELETE /api/processes/:id` and refresh the list.
- Reuse `ProcessForm` for editing existing stopped processes (call `PUT /api/processes/:id`).

### 4. Build and Production Verification
- Run `npm run build` to generate `web/dist`.
- Start `rtx serve` (ensure it serves the `web/dist` folder correctly).
- Verify the entire lifecycle from a single production-ready binary.

## Verification Strategy

### Automated Tests
- None planned for the UI prototype beyond build checks.

### Manual Verification
- Start `rtx serve`.
- Open the UI.
- Create two processes (A and B where B depends on A).
- Start B and verify that A starts automatically (if dependency logic from Phase 7 works as intended).
- Open the log viewer for a running process and verify that new logs appear without a page refresh.
- Stop a process and verify it transitions to `stopped` in the UI.
- Delete a process and verify it is removed from the list.
