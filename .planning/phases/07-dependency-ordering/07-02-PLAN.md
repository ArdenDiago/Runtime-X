**Add to `internal/scheduler/deps.go`:**

1. Add sentinel error:
   `var ErrDependencyNotReady = errors.New("dependency not running")`

2. Implement `waitRunning(s *Scheduler, name string, timeout time.Duration) error`:
   - Polls `s.mu.RLock()` -> check `mp.State == StateRunning` -> `s.mu.RUnlock()` in a loop with 10ms sleep.
   - Also check for terminal states (StateFailed, StateStopped) — if the process died while waiting, return an error immediately instead of waiting for timeout.
   - If deadline exceeded, return descriptive error with process name and timeout.

3. Implement `StartAll() error` on `*Scheduler`:
   - Acquire `s.mu.RLock()`, snapshot `s.processes` into a local map copy, release lock.
   - Call `topoOrder(processesCopy)`. Return error if it fails.
   - If layers is empty (no processes), return nil.
   - For each layer: start all processes in the layer by calling `s.Start(name)`. If any Start fails, return error with context.
   - After starting all processes in a layer, call `waitRunning(s, name, 10*time.Second)` for each process in that layer. If any fails to reach Running, return error.
   - Return nil on success.

4. Add dependency-readiness check to `Start()` in `internal/scheduler/deps.go` as a helper function `checkDepsRunning(processes map[string]*ManagedProcess, def ProcessDef) error`:
   - For each name in `def.DependsOn`, check that `processes[name].State == StateRunning`.
   - If any dependency is not Running, return `fmt.Errorf("%w: %s requires %q to be running (current state: %s)", ErrDependencyNotReady, def.Name, dep, processes[dep].State)`.

   ```go
       }
   }
   ```
   This goes AFTER the state switch (we already confirmed this process CAN start) and BEFORE transition(mp, StateStarting).
