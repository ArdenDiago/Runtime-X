**Create `internal/scheduler/deps.go`:**

1. Add sentinel errors:
   - `var ErrDependencyCycle = errors.New("dependency cycle detected")`
   - `var ErrDependencyNotFound = errors.New("dependency not found")`

2. Implement `topoCheck(processes map[string]*ManagedProcess, newDef ProcessDef) error`:
   - First loop: for each name in `newDef.DependsOn`, check self-reference (`dep == newDef.Name` returns `ErrDependencyCycle` with message), then check existence in `processes` map (missing returns `ErrDependencyNotFound` with message including the missing name).
   - Build in-degree map and adjacency list (dependents) from ALL existing processes PLUS the new definition.
   - Run Kahn's BFS: queue starts with in-degree == 0 nodes, drain and decrement dependents.
   - If `processed < len(processes) + 1`, return `ErrDependencyCycle`.
   - Follow the exact algorithm from 07-RESEARCH.md Pattern 1.

3. Implement `topoOrder(processes map[string]*ManagedProcess) ([][]string, error)`:
   - Build in-degree map and adjacency list from all processes.
   - Layer-based Kahn's: each iteration collects all zero-in-degree nodes as a layer, `sort.Strings(layer)` for determinism, delete processed nodes from in-degree map, decrement dependents.
   - If any nodes remain in in-degree map after draining, return `ErrDependencyCycle` (defensive).
   - Return `[][]string` where each inner slice is a start layer.
   - Handle empty map: return `nil, nil`.
   - Follow the exact algorithm from 07-RESEARCH.md Pattern 2.

**Modify `internal/scheduler/scheduler.go` Register():**

4. After the duplicate name check (`if _, exists := s.processes[def.Name]; exists { ... }`), add:

   ```go
   // Phase 7: validate dependency edges before accepting the definition.
   if len(def.DependsOn) > 0 {
       if err := topoCheck(s.processes, def); err != nil {
           return err
       }
   }
   ```
