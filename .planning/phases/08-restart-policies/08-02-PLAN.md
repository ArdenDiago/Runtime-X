# Plan: Phase 08-02 - Restart Logic and Integration

## Goal
Implement the backoff wait, automatic restart, and stop-cancellation logic.

## Tasks

### 1. Implement `waitAndRestart` (RST-02)
- Create `internal/scheduler/restart.go`.
- Define unexported `waitAndRestart(s *Scheduler, mp *ManagedProcess)`:
  - Calculate `delay = min(Delay * (Factor ^ RestartCount), MaxDelay)`.
  - Use `time.NewTimer(delay)`.
  - Select on `timer.C` and `mp.restartCancelCh`.
  - If `timer.C` fires, call `s.Start(mp.Def.Name)`.
  - If `restartCancelCh` fires, exit the goroutine.

### 2. Update `monitorProcess` (RST-01, RST-03)
- Edit `internal/scheduler/lifecycle.go`.
- After process exits:
  - Check `mp.Def.RestartPolicy`.
  - If policy is `always` or (`on-failure` and `ExitCode != 0`):
    - Check if `RestartCount < MaxRetries` or `MaxRetries == 0`.
    - If retries left:
      - Increment `RestartCount`.
      - Transition to `StateRestarting`.
      - Launch `go waitAndRestart(s, mp)`.
    - Else:
      - Transition to `StateFailed`.
  - Else:
    - Normal transition to `StateStopped` or `StateFailed`.

### 3. Update `Stop()` (RST-04)
- Edit `internal/scheduler/lifecycle.go`.
- In `Stop()` method:
  - If `mp.State == StateRestarting`:
    - Transition to `StateStopping`.
    - Close `mp.restartCancelCh` to stop the wait.
    - Transition to `StateStopped`.
    - Return immediately (no need to signal `SIGTERM` since it's already stopped).

### 4. Update `Start()` (RST-04)
- Initialize `restartCancelCh` on first start.
- Ensure `RestartCount` is NOT reset on manual `Start()` but reset on `Remove()`? Actually, requirements don't specify when to reset `RestartCount`. Let's assume it resets only when the process is removed and re-added.
