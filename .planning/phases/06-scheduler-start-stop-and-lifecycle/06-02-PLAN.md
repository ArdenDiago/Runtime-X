Replace the Stop() stub with the full implementation. Follow the locked patterns from CONTEXT.md exactly:

**Stop(name string) error:**
1. Acquire s.mu.Lock()
2. Look up process by name — return ErrNotFound (wrapped) if missing
3. Check state:
   - StateRunning: proceed
   - StateStopped, StateIdle, StateFailed: unlock, return ErrNotRunning (wrapped with state context)
   - StateStarting, StateStopping: unlock, return descriptive error (cannot stop in transient state)
4. Transition to StateStopping via transition(mp, StateStopping)
5. Create doneCh: `doneCh := make(chan struct{})`; assign `mp.doneCh = doneCh`
6. Capture local copies: `pid := mp.cmd.Process.Pid`, `timeout := mp.Def.StopTimeout`
7. If timeout <= 0, default to 5*time.Second
8. s.mu.Unlock()
9. Send SIGTERM to process group: `syscall.Kill(-pid, syscall.SIGTERM)` (ignore error — process may have just exited)
10. Select:
    - `case <-doneCh:` — process exited within timeout, return nil
    - `case <-time.After(timeout):` — escalate: `syscall.Kill(-pid, syscall.SIGKILL)`, then `<-doneCh` (SIGKILL always works), return nil

**Key coordination with monitorProcess (from Plan 01):**
- Stop() creates doneCh and assigns to mp.doneCh while holding the write lock
- monitorProcess (running in its goroutine) calls cmd.Wait(), then acquires the write lock, updates state, and closes doneCh if non-nil
- This is safe because both operations are serialized by the write lock — doneCh is always assigned before the signal is sent, and the monitor checks it after acquiring the lock

**Verify the interaction with monitorProcess:** The monitor goroutine from Plan 01 should already handle the doneCh pattern:
- After cmd.Wait() returns, it locks, sets state, and if mp.doneCh != nil, closes it and nils it
- If Stop() races with natural exit: Stop() holds the lock while creating doneCh → unlocks → sends signal → blocks on doneCh. Meanwhile, monitor already had cmd.Wait() return, tries to lock, gets it after Stop() unlocks, closes doneCh, Stop() unblocks.
- If natural exit happens before Stop() runs: mp.State will already be Stopped/Failed when Stop() checks — Stop() returns ErrNotRunning immediately.

Run the full test suite: `go test -race -v ./internal/scheduler/...` — all tests (Plan 01 + Plan 02) must pass with zero data races.
