# Plan: Phase 10-01 - CLI Subcommands and `rtx serve`

## Goal
Refactor the CLI to support subcommands and implement `rtx serve` to start the REST API and serve the frontend.

## Context
Phase 10 depends on Phase 9 (REST API). We assume `internal/api/server.go` exists and provides a way to start the server.

## Tasks

### 1. Refactor `cmd/rtx/main.go` for Subcommands
- Refactor `run()` in `main.go` to use `flag.NewFlagSet` for `run` and `serve`.
- Isolate existing `run` logic into a `cmdRun(args []string)` function.
- Ensure global flags (like `-v`/`--verbose`) are still handled before subcommands.
- Maintain backward compatibility for `rtx run <command> [args...]`.

### 2. Implement `rtx serve` Logic
- Create `cmd/rtx/serve.go` (or add to `main.go` if small).
- Implement `cmdServe(args []string)` function.
- `cmdServe` should:
    - Parse flags (e.g., `-port`, default 8080).
    - Instantiate a `scheduler.Scheduler`.
    - Instantiate an `api.Server` with the scheduler.
    - Start the HTTP server.
- Serve the React frontend from `web/dist` at the root path using `http.FileServer`.
- Ensure the API remains accessible under `/api/`.

## Verification Strategy

### Automated Tests
- Since `main.go` is hard to unit test, focus on manual verification and integration tests if possible.
- Verify that `rtx run` still works as expected (backward compatibility).

### Manual Verification
- `go build -o rtx ./cmd/rtx`
- `./rtx run echo "hello"` -> should print hello and exit.
- `./rtx serve -port 9000` -> should start server on port 9000.
- `curl http://localhost:9000/api/processes` -> should return empty list (200 OK).
- Create a dummy `web/dist/index.html` and verify `curl http://localhost:9000/` returns it.
