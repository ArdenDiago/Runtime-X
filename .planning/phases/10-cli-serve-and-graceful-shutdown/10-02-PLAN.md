# Plan: Phase 10-02 - Graceful Shutdown and `StopAll()`

## Goal
Implement a clean shutdown sequence for `rtx serve` that stops all managed processes before the server exits.

## Context
Phase 10 depends on Phase 9. `scheduler.Stop()` must be robust.

## Tasks

### 1. Implement `scheduler.StopAll()`
- Add `StopAll()` method to `internal/scheduler/lifecycle.go` (or `deps.go` if preferred).
- `StopAll()` should:
    - Get a list of all registered processes.
    - Start a goroutine for each process in `StateRunning` or `StateStarting` that calls `Stop(name)`.
    - Use a `sync.WaitGroup` to wait for all `Stop()` calls to complete.
- This ensures all processes are terminated in parallel with their own timeouts and SIGKILL escalations.

### 2. Implement Signal Handling in `rtx serve`
- Refactor `cmdServe` to listen for `SIGINT` and `SIGTERM` using `os/signal`.
- On receiving a signal, initiate the shutdown sequence:
    1. Create a context with a 10-second timeout for graceful shutdown.
    2. Call `httpServer.Shutdown(ctx)` to stop accepting new requests and drain existing ones.
    3. Call `scheduler.StopAll()` to ensure all managed processes are terminated.
    4. Wait for both to complete or for the 10-second timeout to expire.
    5. Log completion and exit the CLI with code 0.

## Verification Strategy

### Automated Tests
- Create `internal/scheduler/lifecycle_test.go` (if not exists) and add `TestStopAll()`.
- `TestStopAll()` should:
    - Register multiple processes.
    - Start them all.
    - Call `StopAll()`.
    - Verify that all processes are in `StateStopped` or `StateFailed` (terminal states).
    - Verify that no child processes remain.

### Manual Verification
- `./rtx serve`
- Register several processes via the API (using `curl`).
- Start them all via the API.
- Press `Ctrl+C`.
- Observe logs indicating the server is shutting down and stopping processes.
- Run `ps aux | grep <process_name>` to verify no orphans remain.
